# Contributing to QuestKit

Thanks for your interest in QuestKit! This document covers what you need to know
to get a working local checkout, run the tests, and open a useful pull request.

## Welcome

QuestKit is an early-stage, public, open-source project. Bug reports, feature
ideas, docs improvements, and code contributions are all welcome. If you are
unsure whether a change is in scope, open an issue first and we can talk it
through before you spend time on a PR.

By participating, you agree to abide by our [Code of Conduct](./CODE_OF_CONDUCT.md).

## Prerequisites

- **Node 20.x** ‚Äî pinned in [`.nvmrc`](./.nvmrc). Use `nvm use` (or your
  equivalent) so your version matches CI.
- **pnpm 10.27.0** ‚Äî declared in `package.json` `packageManager`. Enable it
  with `corepack enable && corepack prepare pnpm@10.27.0 --activate`.
- **Git** ‚Äî recent version with hook support (Husky installs a pre-commit
  hook on `pnpm install`).
- **(Optional) Wrangler** ‚Äî for any task that touches Cloudflare Workers
  locally. Installed as a workspace dev dep, no global install required.

## Clone & install

```bash
git clone https://github.com/ilGentEAcutoO/QuestKit.git
cd QuestKit
pnpm install
```

The first `pnpm install` will also register the Husky pre-commit hook (via the
`prepare` script).

## Common workflow

```bash
pnpm dev        # turbo dev ‚Äî run watch tasks across the workspace
pnpm test       # turbo test ‚Äî run unit + worker test suites
pnpm lint       # ESLint flat config (@antfu/eslint-config)
pnpm typecheck  # tsc --noEmit per package/worker
pnpm build      # production builds for everything publishable
```

You can scope any of these to a single package using turbo's `--filter`:

```bash
pnpm --filter @questkit/core test
pnpm --filter questkit-worker-api dev
```

## Commit conventions

We follow [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/).
Examples:

- `feat(react): add MissionCard component`
- `fix(api): reject expired JWTs in middleware`
- `chore: bump dependabot deps`
- `docs: tighten quick-start example`

Per the project rule in `CLAUDE.md` ¬ß7, commits must **not** contain AI
attribution lines such as "Generated by Claude". Keep messages plain and human.

## Pull request process

1. Fork the repo and create a feature branch from `main`.
2. Make one logical change per PR ‚Äî small, reviewable diffs merge faster.
3. Run `pnpm lint`, `pnpm typecheck`, and `pnpm test` locally before pushing.
4. Open a PR using the template. Link any related issue with `Closes #N`.
5. CI must be green ‚Äî lint, typecheck, tests, gitleaks, and audit all run on
   each PR.
6. A reviewer will leave comments. Address them with follow-up commits (we
   squash on merge, so don't worry about cleaning history yourself).

## Reporting bugs and requesting features

Use the issue templates in `.github/ISSUE_TEMPLATE/`. Include reproduction
steps, expected vs. actual behaviour, and environment details (Node version,
OS, browser if relevant).

## Security

Please do **not** open public issues for vulnerabilities. See
[`SECURITY.md`](./SECURITY.md) for the responsible-disclosure process.
