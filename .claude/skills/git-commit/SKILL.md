# Git Commit

Create professional, clean git commits.

## Commit Message Format

```
<type>: <description>    (≤72 chars)

[optional body]
[optional footer]
```

### Types (if project uses conventional commits)

- `feat:` New feature
- `fix:` Bug fix
- `refactor:` Code restructuring
- `docs:` Documentation
- `style:` Formatting, no logic change
- `test:` Adding/updating tests
- `chore:` Build, deps, config

### Rules

1. **≤72 characters** for subject line
2. **Imperative mood:** "Add", "Fix", "Update" (not "Added", "Fixes")
3. **No AI signatures** — NEVER include "Generated by Claude", "Co-authored-by: Claude", or any AI attribution
4. **No emoji** unless project convention
5. **Meaningful description** — what changed and why

### Examples

```
feat: add user authentication with JWT
fix: resolve race condition in payment processing
refactor: extract validation logic into composable
chore: upgrade nuxt to 3.15
```

## Workflow

```bash
# 1. Check changes
git status
git diff --stat

# 2. Stage (prefer specific files)
git add src/components/LoginForm.vue
# or: git add -A

# 3. Commit
git commit -m "feat: add login form with validation"

# 4. If "commit and push" → trigger git-push skill
```

## Auto-detect convention

```bash
git log --oneline -10
# If conventional commits pattern → use it
# If not → use simple descriptive messages
```
